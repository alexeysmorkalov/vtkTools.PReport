PReport
 

The report template contains the unlimited number of pages (base class - TprCustomPage). Each page contains a set of bands (base class of band - TprCustomBand). The band contains objects (base class - TprObj). Each object contains two parameters records (base class - TprObjRec), one parameters record is used when the report are designed and other when the report are generating. Each parameters record contains the list of the views (base class TprObjRecVersion), for each view the formula is specified, that specifies when (under what conditions) a view can be used. Also TprObjRec contains index of default view, which shows at designer and is used by default.

The report generating happens in three passes.

  • Each TprCustomPage object generates the table in which rows are formed by horizontal bands and columns by vertical bands. If the report template does not contains vertical bands the table are formed only from rows. All views of objects are generating because on first pass not always it is possible to select the necessary view, for example if in the formula are used the group summary.
  • The generated table divides by the end pages (objects of the TprCustomEndPage class), which forms a prepared report.
  • All aggregate values are calculated, for each object the necessary view is selected and these views are transferred on TprCustomEndPage.
