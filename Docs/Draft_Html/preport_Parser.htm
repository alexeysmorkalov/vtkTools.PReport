PReport
 

PReport has the embedded expressions interpreter. From expressions you can access to the application components and their properties. The PReport interpreter does not support RTTI properties, it supports limited list of properties that are described below. The interpreter supports data types: String, Boolean, Integer, DateTime, and Double. Also is introduced new data type – YearMonth this type represents the integer of the format: YYYYMM where:

  • YYYY – year
  • MM – month
  • For example: 200312 – December of 2002 year.

For access to the objects methods and properties the point is used, for example: Table1.Eof() or Table1.RecordCount. For access to properties and methods of the report you can not write a name of the report; simply write a name of a property or method. The order of search of components:

  1. Within the report component (TprReport or TprTxReport) (FindComponent method is used).
  2. Within the owner of the report component (if it not nil), i.e. it is the form or DataModule in which the report places. (FindComponent method is used)
  3. Within all datamodules of application. (Screen.DataModules).
  4. Within all forms of application (Screen.Forms). You may provide your list of the available components, use OnGetAvailableComponents event of the TprCustomReport component for that.
