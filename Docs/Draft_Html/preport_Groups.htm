PReport
 

Groups used for grouping data by some criteria. You can use any PReport expression as group condition (usually used expression based on DB fields). When this expression changes, PReport starts new group. Groups are represented by the TprGroup class. For each group three properties must be specified:

  • The group name. Must be unique in the report.
  • The detail band (vertical or horizontal) to which group are linked.
  • The group expression. Can be any PReport expression, usually expression are based on DB fields. Examples of group expressions:
    • Table1.CustomerName – data will be grouped by the CustomerName DB field.
    • Copy(Table.Name, 1, 1) – data will be grouped by the first letter of the Name DB field.

      • If the report template has enclosed groups, the subgroup expression must include expression of the parent group. For example: the dataset (Query1) based on this query:
      select * from customer order by country, name
      This dataset linked to the detail band named Detail. The report must be grouped by the country and first letter of the customer name. You must create two groups:
      • First group:
        • The group name – ContryGroup
        • The detail band – Detail
        • The group expression – Query1.Country.
      • Second group:
        • The group name – NameGroup
        • The detail band – Detail
        • The group expression – UID(Query1.Country, Copy(Query1.Name, 1, 1))
      The UID function, accepts one and more input parameters and returns a string formed by concatenation of input parameters (if a parameter not a string, it is converted to a string). The strings are divided by a numeral with a code #01 (0x01).
